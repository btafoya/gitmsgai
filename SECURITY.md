# Security Policy

## Security Features

The GitMsgAI VSCode extension implements multiple layers of security to protect your sensitive data:

### 1. Secure API Key Storage

- **VS Code SecretStorage API**: Your OpenRouter API key is stored using VS Code's SecretStorage API, which provides OS-level encryption
- **Platform-specific security**:
  - Windows: Credential Manager
  - macOS: Keychain
  - Linux: Secret Service API (libsecret)
- **No plain text storage**: API keys never appear in `settings.json` or any configuration files
- **Automatic migration**: If you previously stored your API key in settings, use the `GitMsgAI: Set OpenRouter API Key` command to migrate to secure storage

### 2. Rate Limiting

- **Sliding window rate limiter**: Prevents accidental API abuse and excessive costs
- **Default limit**: 10 requests per minute (configurable)
- **User-friendly errors**: Clear messages when rate limit is reached, including countdown timers
- **Protection against**: Accidental button mashing, automation scripts, infinite loops

### 3. Sensitive File Exclusion

The extension automatically filters out sensitive files from git diffs before sending to the AI:

**Default exclusion patterns**:
- `.env*` - Environment variable files
- `*.key` - Key files
- `*.pem`, `*.p12`, `*.pfx` - Certificate files
- `credentials.json` - Credential files
- `secrets.*`, `*.secret` - Secret files
- `**/secrets/**` - Files in secrets directories

**Configuration**:
```json
{
  "gitmsgai.excludePatterns": [
    ".env*",
    "*.key",
    "custom-secret-*.json"
  ],
  "gitmsgai.warnOnSensitiveFiles": true
}
```

When sensitive files are detected:
- They are automatically excluded from the diff
- A warning notification is shown (if enabled)
- Lists which files were excluded

### 4. Input Validation & Sanitization

**API Response Validation**:
- Validates response structure from OpenRouter API
- Checks for required fields (choices, message, content)
- Handles malformed or unexpected responses gracefully

**Commit Message Sanitization**:
- Removes control characters and escape sequences
- Limits message length (max 2000 characters)
- Strips dangerous shell characters
- Validates UTF-8 encoding

**Error Message Sanitization**:
- Strips API keys from error messages (masks with `***`)
- Removes sensitive file paths
- Sanitizes diff content in error output
- Prevents information leakage through error logs

### 5. Request Timeout & Cancellation

- **Configurable timeout**: Default 30 seconds (prevents hanging requests)
- **User cancellation**: Click cancel in progress notification to abort requests
- **Network failure handling**: Graceful error messages for connection issues

### 6. User Consent & Transparency

**First-run consent warning**:
- Explains what data is sent to OpenRouter
- Can be disabled via `gitmsgai.showConsentWarning: false`
- Provides link to privacy documentation

**Transparency**:
- Clear documentation of what data is transmitted
- Open source code for full audit
- No telemetry or analytics collection

## Data Privacy

### What Data is Sent to OpenRouter

When you generate a commit message, the extension sends **only** the following to OpenRouter's API:

1. **Git diff of staged changes**: The code differences (added/removed lines)
2. **Your prompt template**: The configured prompt with `{changes}` replaced by the diff
3. **Model selection**: The AI model name you've configured

**What is NOT sent**:
- Your name or email
- Git history or commit logs
- Unstaged changes
- Files matching exclusion patterns (e.g., `.env` files)
- Your API key (only sent in Authorization header, not in request body)
- Personal information
- Telemetry or usage statistics

### Example of Data Sent

```json
{
  "model": "mistralai/mistral-7b-instruct",
  "messages": [
    {
      "role": "user",
      "content": "Given these staged changes:\ndiff --git a/src/app.ts...\n\nGenerate a commit message..."
    }
  ]
}
```

### What Data is Stored Locally

| Data Type | Storage Location | Encryption | Retention |
|-----------|-----------------|------------|-----------|
| API Key | VS Code SecretStorage | OS-level encryption | Until manually removed |
| Cache | Workspace State | None (commit messages only) | Until cleared or cache size exceeded |
| Settings | VS Code Settings | None (no secrets) | Until manually changed |
| Consent Flag | Workspace State | None | Permanent (unless workspace deleted) |

**Cache contents**: Only commit messages generated by the AI (no code or diffs)

**Cache security**:
- Stored per-workspace (not shared between projects)
- Automatically rotates when size limit reached (default: 10 entries)
- Can be manually cleared via command palette

## What Data is NOT Stored

- Your code or git diffs (only used in memory, never persisted)
- API request/response logs
- Personal information
- Usage statistics
- Error logs containing sensitive data

## Third-Party Services

### OpenRouter

This extension uses [OpenRouter](https://openrouter.ai/) to access various AI models.

**Data sent to OpenRouter**:
- Git diff (staged changes only)
- Your prompt template
- Model selection

**OpenRouter's responsibilities**:
- They process your request and return a commit message
- Review their [Privacy Policy](https://openrouter.ai/privacy)
- Review their [Terms of Service](https://openrouter.ai/terms)

**Your responsibilities**:
- Ensure you have the right to send code to external services
- Review your organization's policies on AI tool usage
- Use exclusion patterns to prevent sending sensitive files

## Security Best Practices

### For Users

1. **Use the secure API key command**:
   ```
   GitMsgAI: Set OpenRouter API Key
   ```
   Don't store API keys in settings.json

2. **Review exclusion patterns**:
   - Ensure sensitive files are excluded
   - Add custom patterns for proprietary file types
   - Test with dummy changes first

3. **Enable review mode** for sensitive repositories:
   ```json
   {
     "gitmsgai.reviewBeforeApply": true
   }
   ```

4. **Monitor your OpenRouter usage**:
   - Check your OpenRouter dashboard regularly
   - Set up usage alerts
   - Use rate limiting to prevent excessive API calls

5. **Keep the extension updated**:
   - Updates may include security fixes
   - Review changelogs for security improvements

6. **Use caution in corporate environments**:
   - Check your organization's policies on AI tools
   - Consider using private AI endpoints if available
   - Don't commit sensitive business logic without approval

### For Developers & Contributors

1. **Never log sensitive data**:
   - No API keys in console.log
   - Sanitize diffs before logging
   - Use safe error messages

2. **Validate all inputs**:
   - API responses
   - User configurations
   - File paths and patterns

3. **Follow least privilege principle**:
   - Only request necessary VS Code permissions
   - Minimize data sent to external services

4. **Security review for changes**:
   - Any code touching API keys, diffs, or external requests
   - Changes to validation or sanitization logic
   - New configuration options

5. **Keep dependencies updated**:
   - Regularly audit npm dependencies
   - Use `npm audit` to check for vulnerabilities
   - Update to patched versions promptly

## Reporting Security Vulnerabilities

We take security vulnerabilities seriously. If you discover a security issue, please follow responsible disclosure:

### DO:

1. **Email the maintainer directly**: [Create a private issue or email from the GitHub profile]
2. **Provide detailed information**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)
3. **Give us time to fix**: Please allow 90 days for a patch before public disclosure
4. **Coordinate disclosure**: Work with us on timing and disclosure details

### DO NOT:

- Post security vulnerabilities in public issues
- Exploit vulnerabilities to access data you don't own
- Test against production systems without authorization
- Demand ransom or compensation

### What to Expect:

1. **Acknowledgment**: Within 48 hours
2. **Initial assessment**: Within 5 business days
3. **Fix timeline**: Varies by severity, but typically:
   - Critical: 7-14 days
   - High: 30 days
   - Medium: 60 days
   - Low: 90 days
4. **Credit**: We'll acknowledge your contribution in the security advisory and CHANGELOG

### Security Advisory Process:

1. Vulnerability reported via private channel
2. Maintainers assess severity and impact
3. Patch developed and tested
4. Security advisory published (if severity >= medium)
5. Patch released
6. Public disclosure coordinated with reporter

## Compliance & Regulations

### GDPR Compliance

- No personal data is collected or processed by the extension
- API key storage uses OS-level encryption
- Users control what data is sent to OpenRouter
- Data retention is transparent and configurable

### Corporate Compliance

If using in a corporate environment:

1. Review your organization's policies on:
   - AI tool usage
   - External API access
   - Code sharing with third parties

2. Consider:
   - Using private AI endpoints
   - Implementing additional exclusion patterns
   - Restricting usage to non-proprietary code
   - Requiring review mode for all commits
